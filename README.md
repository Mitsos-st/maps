## Σύντομη περιγραφή
Το jabbamaps.c είναι ένα πρόγραμμα σε γλώσσα C που χρησιμοποιεί τον αλγόριθμο Held-Karp
για την επίλυση του Traveling Salesman Problem (χωρίς την προϋπόθεση να γυρίσει πίσω στην αρχική πόλη).
Το πρόγραμμα δέχεται ένα αρχείο με τις πόλεις που θα επισκεφτεί ο χρήστης και τις αποστάσεις μεταξύ
των πόλεων και υπολογίζει τη βέλτιστη διαδρομή με το ελάχιστο κόστος.
Εμφανίζει την ιδανική σειρά επισκέψεων στις πόλεις και το συνολικό
κόστος της διαδρομής. Το πρόγραμμα υποστηρίζει δυναμικό προγραμματισμό
για αποτελεσματική επεξεργασία. Η χρήση του αλγορίθμου held-karp μειώνει κατα πολύ την πολυπλοκότητα απο n! που θα ήταν ο έλγχος όλων των πιθανών διαδρομών σε n<sup>2</sup> · 2<sup>n</sup> .
Όμως έχει μεγάλη χωρική πολυπλοκότητα n · 2<sup>n</sup> .

### Συνάρτηση number_city
Η συνάρτηση αυτή παίρνει ως όρισμα έναν πίνακα από struct data
```c
struct data{
    int distance;
    char *city1;
    char *city2;
}
``` 
όπου το κάθε struct έχει αποθηκευεμένη μία γραμμή του αρχείου.
Και η συνάρτηση επιστρέφει έναν πίνακα από struct number_city
```c
struct number_city{
    int city1_number;
    int city2_number;
    int distance;
}
```
Ο στόχος της συνάρτησης είναι να αντιστοιχίσει κάθε πόλη σε έναν αριθμό.
```

```

### Πίνακας distance
Οι μεταξύ αποστάσεις των πόλεων αποθηκεύονται σε έναν πίνακα έτσι ώστε να αποφευχθεί η χρήση μιας συνάρτησης distance η οποία θα έπρεπε να κλειθεί πολλές φορές και θα έπαιρνε πολύ χρόνο το πρόγραμμα.
- Οι πόλεις αποθηκεύτηκαν ως εξής:
```c
int distance[n*100+n];
for(int i=0;i<size;i++){
distance[city[i].city1_number*100+city[i].city2_number]=city[i].distance;
distance[city[i].city2_number*100+city[i].city1_number]=city[i].distance;
}
```
Οι πόλεις κωδικοποιήθηκαν σε έναν τετραψήφιο αριθμό, με δύο ψηφία να αποτελεί κάθε πόλη.Εφόσον ο μέγιστος αριθμός των πόλεων είναι 64 ο τετραψήφιος αριθμός είναι αρκετός για την κωδικοποίηση.  
Π.χ. 
- Η απόσταση της 5ης και της 3ης πόλης βρίσκεται στα στοιχεία distance\[0503\] και distance\[0305\].

### Συνάρτηση Held_karp_function
Η συνάρτηση αυτή με τη χρήση του αλγόριθμου Held-Karp
βρίσκει τη βέλτιστη διαδρομή.
Η συνάρτηση χρησιμοποιεί τρεις πίνακες, τον πίνακα με τις κωδικοποιημένες αποστάσεις και τους δισδιάστατους πίνακες g και pred μεγέθους 2<sup>n</sup> · n όπου n ο αριθμός των πόλεων.
```c
int** g = malloc((1<<n) * sizeof(int*));
int** pred = malloc((1<<n) * sizeof(int*));
if(!g || !pred){
    fprintf(stderr,"Failed to allocate memory\n");
    exit(1);
}
for(int i=0;i<1<<n;i++){
    g[i]=malloc(n * sizeof(int));
    pred[i]=malloc(n * sizeof(int));
    if(!g[i] || !pred[i]){
        fprintf(stderr,"Failed to allocate memory\n");
        exit(1);
    }
}
```
Ο πίνακας g\[S\]\[K\] παίρνει ένα σύνολο S ως πρώτο στοιχείο και ένα αριθμό k ως δεύτερο.
1. Το σύνολο S καθορίζει τις πόλεις που έχουμε περάσει
- π.χ. Το S={2,3,5} δείχνει ότι έχουμε περάσει από τις πόλεις 2,3,5 .
- Στο πρόγραμμα το S συμβόλιζεται με έναν αριθμό και οι `Άσοι` στη δυαδική μορφή του αριθμού δείχνουν τις πόλεις. Π.χ. S = 5<sub>10</sub> = 101<sub>2</sub> δηλαδή το S=5 είναι το σύνολο {1,3} . 

2. Το K είναι η πόλη που καταλήγουμε
3. Το g δείχνει την βέλτιστη διαδρομή αρχίζοντας από την πόλη 1 περνώντας από τις πόλεις του συνόλου S και καταλήγοντας στην πόλη K .
- Π.χ. Το g\[6\]\[7\]  είναι η βέλτιστη διαδρομή η οποία αρχίζει από την πόλη 1 διασχίζει τις πόλεις 2 και 3 και καταλήγει στην πόλη 7.

Ομοίως για τον πίνακα pred\[S\]\[K\] μόνο που ο pred δείχνει την προτελευταία πόλη όπου K θεωρείται η τελευταία. Ο pred είναι απαραίτητος για να βρεθεί η πλήρης διαδρομή που ακολουθήθηκε.

Το πρόγραμμα ακολουθεί τον παρακάτω ψευδοκώδικα για την εύρεση της βέλτιστης διαδρομής.

```
function algorithm TSP (G, n) is
    for k := 2 to n do
        g({k}, k) := d(1, k)
    end for

    for s := 2 to n−1 do
        for all S ⊆ {2, ..., n}, |S| = s do
            for all k ∈ S do
                g(S, k) := min m≠k,m∈S [g(S\{k}, m) + d(m, k)]
            end for
        end for
    end for

    opt := mink≠1 [g({2, 3, ..., n}, k) + d(k, 1)]
    return (opt)
end function
```
Αφού βρεθεί το βέλτιστο κόστος πάμε προς τα πίσω με τον πίνακα pred για να βρεθεί η σειρά των πόλεων και οι πόλεις αποθηκεύονται αντίστροφα σε έναν πίνακα.
```c
//Γυρίζοντας προς τα πίσω απο την τελευταία
//Βρίσκει όλη τη σειρά των πόλεων
int minus=0;
int array[n]; //Πίνακας για την αποθήκευση της σειράς των πόλεων
array[0]=temp; //Η πρώτη τιμή είναι η τελευταία πόλη
//Αφαιρώντας μία μία τις πόλεις απο το σύνολο των πόλεων βρίσκει τη σείρα
for(int k=0;k<n-2;k++){
    temp2=pred[(1<<n)-2-minus][temp-1];
    array[k+1]=temp2;
    minus+=(1<<(temp-1));
    temp=temp2;
}
array[n-1]=1; //Η τελευταία τιμή είναι η πρώτη πόλη
//Εύρεση των ονομάτων των πόλεων της βέλτιστης διαδρομής
```
#### Σημαντική σημείωση 
Οι πόλεις είναι αριθμημένες με αρχή την πόλη 1 αλλά στους πίνακες g και pred το στοιχείο Κ που δείχνει την τελική πόλη έχει ως πρώτη πόλη την 0 οπότε ο κώδικας αφαιρεί 1 από το στοιχείο Κ .

### Συνάρτηση find_city_names
Η συνάρτηση αυτή παίρνει ως ορίσματα τον πίνακα από struct city_connection ο οποίος συνδέει τα ονόματα των πόλεων με τους αριθμούς και τον πίνακα που έχει αποθηκευμένη τη σειρά των πόλεων αντίστροφα.
Η συνάρτηση συνδιάζει τους παραπάνω πίνακες και επιστρέφει έναν πίνακα που έχει τη σειρά των πόλεων αντίστροφα, αλλά ως ονόματα αντί για αριθμούς.
Έτσι συνολικά συνδιάζοντας όλα τα παραπάνω τυπώνεται η βέλτιστη διαδρομή και το συνολικό κόστος.

## Σχόλια
- Στην main πριν την αποθήκευση του αρχείου στον πίνακα struct data βρίσκουμε το μέγιστο όνομα πόλης και δεσμεύουμε κατάλληλη μνήμη γαι την αποθήκευση των ονομάτων των πόλεων έτσι ώστε το πρόγραμμα να διαχειρίζεται τις πόλεις όσο μεγάλα ονόματα και αν έχουν.
- Επειδή δεν επιτρεπόταν η χρήση της `math.h` χρησιμοποιήθηκε bit shift για τις δυνάμεις του 2 και φτιάχτηκε η συνάρτηση my_sqrt για την εύρεση της ρίζας των τέλειων τετραγώνων.
- Κατά την ανάγνωση του αρχείου μετράμε και το πλήθος των γραμμών του, εφόσον το αρχείο είναι σωστό το πλήθος αυτό είναι το πλήθος όλων των δυνατών αποστάσεων.
Έτσι βρίσκουμε το πλήθος των πόλεων με τον ακόλουθο τύπο:  
    ### $n = \frac{1+\sqrt{1+8 \cdot i}}{2}$
1. Όπου n οι πόλεις
2. Όπου i το πλήθος των δυνατών αποστάσεων

    - Απόδειξη:  
Το πλήθος όλων των δυνατών αποστάσεων μεταξύ n πόλεων είναι το άθροισμα :  n - 1 + n - 2 + ...  
Αφού η πρώτη πόλη έχει n - 1 απόστασεις από τις υπόλοιπες  
η δεύτερη πόλη έχει n - 2 αποστάσεις από τις υπόλοιπες  
και ούτο κάθε εξής.  
Έτσι :  
    ### $i = \displaystyle\sum_{k=1}^{n-1} k = \frac{n \cdot (n-1)}{2}$  
    Λύνοντας ως προς n:  
    $n^2-n-2\cdot i=0$  
    $Δ = 1 + 8 \cdot i$  
    Απορρίπτοντας την αρνητική λύση καταλήγουμε:  
    $n = \frac{1+\sqrt{1+8 \cdot i}}{2}$
